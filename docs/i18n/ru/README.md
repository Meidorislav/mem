# MEM: Твой локальный технический оракул

**Перестань искать в интернете решения, которые ты уже однажды нашел.**

MEM — это «второй мозг» разработчика внутри терминала. Он индексирует твой технический опыт: сессии в консоли, фрагменты конфигов и заметки по проектам, делая их мгновенно доступными через локальный приватный ИИ.

---

## 🚀 Основные возможности

*   **`mem ask` (Магия)**: Поиск по твоему опыту на обычном языке. Спроси *«как я чинил 502 ошибку в nginx в прошлом году?»* и получи точную последовательность действий.
*   **`mem watch` (Минимум усилий)**: Записывай терминальную сессию. Реши проблему, а затем используй `mem remember`, чтобы превратить историю команд в готовую инструкцию.
*   **`mem save`**: Мгновенно сохраняй сниппеты, команды-однострочники или заметки по архитектуре.
*   **CLI-First**: Создан для тех, кто живет в терминале. Никакого переключения на тяжелые GUI-приложения.

## 🧠 В чем отличие?

### Смысл, а не просто слова
Обычные инструменты вроде `grep` бесполезны, если ты не помнишь точных слов. MEM использует **семантический поиск**: он понимает *намерение*. Запрос «деплой» легко найдет заметку с заголовком «Настройка сервера».

### Полная приватность
Всё остается на твоем устройстве. MEM использует **локальные векторные эмбеддинги** и **локальную языковую модель (LLM)**. Твой код и конфиги серверов никогда не попадут в облако.

---

## 🛠 Как работает ИИ

1.  **Эмбеддинги**: Твои данные преобразуются в многомерные векторы (математическое представление смысла) с помощью локальной модели.
2.  **Семантический поиск**: Когда ты задаешь вопрос, MEM находит в локальной базе самые близкие по смыслу «воспоминания».
3.  **Локальный синтез**: Локальная LLM анализирует найденный контекст и выдает краткий ответ, основанный исключительно на *твоем* опыте.

## ⚡ Практические примеры

### 1. Сложные команды инфраструктуры
```bash
# Сохраняем последовательность настройки
mem save "Настройка репликации Postgres" -c "apt install postgresql-15; sed -i 's/wal_level = replica/wal_level = logical/' /etc/postgresql/15/main/postgresql.conf"

# Находим через полгода
mem ask "как включить логическую репликацию?"
```

### 2. Команда «Я это точно забуду»
```bash
# Сохраняем хитрую комбинацию find/xargs
mem save "удалить все пустые папки" -c "find . -type d -empty -delete"

# Мгновенно вспоминаем
mem ask "как почистить пустые директории?"
```

### 3. Запись сессии отладки
```bash
# Записываем поток действий при фиксе бага
mem watch
# $ ssh production-server
# $ journalctl -u api-service -n 100
# $ systemctl restart api-service
mem remember "перезапуск api-сервиса на проде"

# Позже спрашиваем, как мы чинили падение
mem ask "как я фиксил падение api-сервиса?"
```
